# Live2DPet 桌面伴侣 Agent 优化方案

本文件基于当前项目实现，给出可落地的运行策略，不需要引入云端中间件。

## 1. 低延迟本地语音识别（已接入）

- Whisper 识别链路已启用以下优化：
1. 录音端自动静音截断（说完即停，不再固定等满时长）。
2. 录音窗口缩短（自动模式约 5.2s，手动模式约 6.8s 上限）。
3. 主进程并发保护（避免重入导致卡顿和 CPU 抢占）。
4. Whisper 可执行与模型路径缓存（减少重复文件探测）。
5. 动态线程与动态超时（按 CPU 与音频时长自适应）。

- 推荐设置：
1. `voiceInput.mode = whisper-stt`
2. `voiceInput.whisperModel = small`（准确率优先）或 `base`（速度优先）
3. 连续监听场景打开 `voiceInput.autoContinuous = true`

## 2. 图像识别更快（已接入）

- 视觉链路已做减载：
1. HQ 截图尺寸从 768 降到 640。
2. JPEG 质量从 40 降到 34。
3. 主请求最多附带 1 张当前截图（由之前多图改为单图优先）。
4. 相同截图短时间内去重，不重复发送视觉 payload。
5. 视觉请求 token 上限单独限制（默认 `visionMaxTokens = 1200`）。

- 说明：
- 这些改动优先降低“上传+推理总时延”，并保持可用识别质量。

## 3. 记忆长期稳定与“近永久”保存（已接入）

- 内存层：
1. 关键词倒排索引，减少全量扫描，提升检索速度。
2. 连续重复消息去重，减少垃圾记忆。
3. 超限时逐条淘汰并同步索引，避免索引漂移。

- 落盘层：
1. 延迟批量保存（debounce）替代每条都写文件。
2. 原子写入（tmp -> rename）降低中断损坏风险。
3. 自动备份 `memories.json.bak`，主文件损坏时可恢复。
4. 手动保存触发时会优先 flush 待写队列。

## 4. 互动反馈更像 Agent（已接入）

- 用户消息链路增强：
1. 用户说话/快捷输入后，先给即时回执气泡（“收到，马上看”）。
2. AI 忙时不会丢消息，最新一条进入队列，当前轮结束后自动处理。
3. 短时间重复输入会抑制，减少重复回复。

## 5. 继续提升为“桌面伴侣 Agent”的建议

以下建议不依赖重构，可逐步做：

1. 任务记忆分层：
- 在现有 `memory` 外增一层 `profile memory`（用户偏好、禁忌、常用工具），仅在变化时更新。

2. 场景意图模板：
- 按窗口类型（IDE/浏览器/视频/游戏）注入不同 system 附加规则，减少模型漂移。

3. 可操作动作回路：
- 为常见动作设计最小 Agent 工具集（如：打开链接、启动程序、复制摘要），每个动作都要求“可撤销”。

4. 观测与回放：
- 记录每次 `语音->识别->请求->回复` 的耗时分段日志（ms），按天查看瓶颈变化。

5. 安全边界：
- 对外部命令和文件写入增加确认策略（高风险动作需二次确认）。

## 6. 建议基线参数

可作为稳定运行基线：

```json
{
  "interval": 2,
  "chatGap": 1,
  "visionMaxTokens": 1200,
  "voiceInput": {
    "mode": "whisper-stt",
    "autoContinuous": true,
    "whisperModel": "small"
  },
  "memory": {
    "enabled": true,
    "autoSave": true,
    "maxMemories": 100000,
    "saveDebounceMs": 1500,
    "minSaveIntervalMs": 3000
  }
}
```
